# GitHub Rulesets Management

## Purpose

Plan and apply approved branch rulesets and required checks.

## Procedure

1. Read the branch delivery policy and required-check plan.
2. Inspect existing repository rulesets read-only.
3. Compare current and desired protection.
4. Draft rules for `develop`, `main` and protected patterns.
5. Include review, required-check and force-push controls.
6. Present the exact remote change for approval.
7. Apply and verify only after approval.

## Evidence

- Produce a GitHub rulesets plan.
- Record existing and resulting rule identifiers.
- Record verification of effective protections.

## Guardrails

- Do not weaken existing protections silently.
- Do not apply remote changes without explicit approval.
- Do not require unstable or missing check names.
