# Project Scaffolding Skill

## Procedure

1. Validate the approved foundation plan, exact paths and command allowlist.
2. Reuse existing project conventions when present.
3. Create only approved directories, package/module declarations and minimal bootstrap.
4. Keep framework code at delivery edges and preserve approved dependency direction.
5. Add only the startup or health behavior required to prove assembly.
6. Run approved compile, startup or smoke commands and record evidence.

## Forbidden

Do not add dependencies, business rules, generic CRUD, complete adapters, sample
features or future module placeholders without separate approval.

## Evidence

Record:

- exact files created or changed;
- approved source for each path;
- commands run and summarized results;
- startup or compile result;
- deferred behavior and dependencies.

## Gates

Stop before dependency changes, infrastructure writes, unknown commands, path changes
or any file not listed in the approved foundation plan.

## Completion

The approved minimal structure exists and starts or compiles without pretending that
unimplemented business behavior is available.
