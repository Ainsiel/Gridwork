# Repository Bootstrap

## Purpose

Create approved framework scaffolds and tooling foundations before domain implementation.

## Procedure

1. Confirm the repository layout and selected stack.
2. Confirm dependency and scaffold commands before execution.
3. Create only approved framework roots and technical boundaries.
4. Add test harness foundations and health or wiring probes.
5. Add root quality-command entry points.
6. Run baseline installation, build and test checks.
7. Record generated paths, commands and decisions.

## Evidence

- Produce a repository bootstrap report.
- Record dependency changes and validation output.
- Hand delivery requirements to `delivery-infrastructure`.

## Guardrails

- Do not implement business use cases or functional screens.
- Do not invent domain modules beyond approved architecture.
- Stop on unapproved dependencies, paths or commands.
