# Next.js Project Bootstrap

## Purpose

Initialize an approved Next.js application boundary without business features.

## Procedure

1. Confirm the frontend path, router model and package manager.
2. Confirm dependency changes and scaffold command.
3. Initialize TypeScript, linting and the approved test harness.
4. Create approved route groups and feature boundaries.
5. Add only minimal health or wiring behavior.
6. Add root quality-command mappings.
7. Run lint, typecheck, tests and production build.

## Evidence

- Record scaffold command, dependencies and created paths.
- Record validation output and unresolved decisions.

## Guardrails

- Do not implement functional screens or business use cases.
- Do not invent routes outside approved architecture.
- Do not generate during Gridwork installation.
